# FPGA Reaction Training System

![Hardware Setup](reaction_training_system.png)

A hardware-based **Reaction Training System** implemented in **Verilog HDL** for an FPGA development board. This project simulates a traffic-light reaction test in which the user must respond as quickly as possible once the green light appears. The system combines multiple digital design concepts—including finite state machines, timers, pseudo-random number generation, PWM, button debouncing, and multiplexed seven-segment displays—into a complete FPGA application.

# Overview

The objective of this project is to create a fully hardware-based reaction training system using Verilog HDL. Unlike software implementations, every function is synthesized directly onto the FPGA, providing deterministic timing and demonstrating the capabilities of digital hardware design.

The system follows a traffic-light sequence where the user initiates a test, waits through a randomized delay, reacts to the green LED, and receives their measured reaction time on the seven-segment display.

# How It Works

1. The user presses the **Start** button.
2. The red LED turns on.
3. The yellow LED remains active for a randomized period generated by the LFSR.
4. The green LED turns on.
5. The reaction timer begins immediately.
6. The user presses the **Reaction** button.
7. The elapsed time is captured.
8. The reaction time is displayed on the seven-segment display.
9. Another test can be started immediately.

# Module Description

## `top.v`

Top-level module responsible for integrating every subsystem of the project. It instantiates and connects the FSM, timers, display controller, buzzer, debouncer, random generator, and LED controller.

---

## `lights_3.v`

Controls the traffic-light LEDs according to the current state of the reaction training system.

---

## `traffic_timers.v`

Generates all timing signals required by the project, including LED durations and reaction time measurement.

---

## `display_system.v`

Drives the multiplexed four-digit seven-segment display and presents the measured reaction time.

---

## `randomGenerator.v`

Implements the pseudo-random delay before the green LED appears.

---

## `THREE_BIT_LFSR.v`

Implements a 3-bit Linear Feedback Shift Register used for pseudo-random number generation.

---

## `THREE_BIT_COUNTER.v`

Counter used together with the LFSR for delay generation.

---

## `TOP_LFSR_COUNTER.v`

Connects the counter and LFSR modules into the complete random timing subsystem.

---

## `buzzer.v`

Generates a PWM signal for the passive buzzer to provide audible feedback.

---

## `debouncing.v`

Removes mechanical bouncing from pushbutton inputs to ensure reliable user interaction.

---

## `constraints.xdc`

Contains the FPGA pin assignments for LEDs, buttons, seven-segment display, buzzer, and clock.

# Building the Project

1. Open **Vivado**.
2. Create a new RTL project.
3. Add all Verilog source files.
4. Add the `constraints.xdc` file.
5. Set `top.v` as the top module.
6. Run **Synthesis**.
7. Run **Implementation**.
8. Generate the **Bitstream**.
9. Program the FPGA.
10. Press the **Start** button to begin the reaction training sequence.
